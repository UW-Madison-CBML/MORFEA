# 脚本是如何运行的？

## 🔍 当前运行方式

你现在是**直接在 CHTC 登录节点（submit node）上运行** Python 脚本：

```bash
# 你直接在命令行运行
python3 generate_tphate_for_aadhitya.py ...
```

### 这种方式的特点：

✅ **优点**：
- 简单直接，立即执行
- 适合小规模测试（几个胚胎）
- 可以实时看到输出

⚠️ **限制**：
- **计算资源有限**：登录节点的 CPU/内存有限
- **可能会被限制**：长时间运行可能被系统终止
- **不能使用 GPU**：登录节点通常没有 GPU
- **影响其他人**：占用共享资源

## 💡 建议：使用 HTCondor 提交作业

对于处理大量数据（704 个胚胎），建议使用 **HTCondor** 提交作业，这样可以：

✅ **优点**：
- 使用专门的计算节点（更多 CPU/内存）
- 可以请求 GPU（如果需要）
- 不会占用登录节点资源
- 可以在后台运行，断开连接也不影响
- 自动管理资源分配
