# KNN (k-nearest neighbors) 参数说明

## 🎯 简单解释

**knn** = **k 近邻**（k-nearest neighbors）

这是 T-PHATE 算法中的一个重要参数，用于决定每个点要"连接"多少个最近的邻居。

## 🔍 比喻理解

想象你在一个派对上：

- **knn = 3**: 每个人只和最近的 3 个人聊天（连接少，图较稀疏）
- **knn = 5**: 每个人和最近的 5 个人聊天（默认，平衡）
- **knn = 10**: 每个人和最近的 10 个人聊天（连接多，图较密集）

在 T-PHATE 中，这个参数控制：
- 每个数据点（胚胎的某个时间点）会连接到最近的 k 个其他时间点
- 这些连接形成一张"邻居图"
- T-PHATE 基于这张图来计算降维结果

## 📊 参数效果

### 较小的 knn (例如 3-5)
- ✅ **优点**: 
  - 计算速度快
  - 更关注局部结构（小范围的相似性）
  - 对噪声更鲁棒
- ⚠️ **缺点**: 
  - 可能丢失一些全局结构
  - 图可能不够连通

### 较大的 knn (例如 10-20)
- ✅ **优点**: 
  - 捕捉更多全局结构
  - 图更连通
  - 平滑度更高
- ⚠️ **缺点**: 
  - 计算时间更长
  - 可能过度平滑，丢失细节
  - 对小数据集可能不合适

### 默认值 knn = 5
- 是一个平衡的选择
- 适合大多数情况
- 对于胚胎轨迹数据通常工作良好

## 🔬 在 T-PHATE 中的具体作用

T-PHATE 使用 knn 来：
1. **构建邻居图**: 每个点连接最近的 k 个点
2. **计算距离/相似度**: 基于这些连接计算点之间的距离
3. **降维**: 将这些关系映射到低维空间（3D）

## 💡 如何选择 knn 值？

### 经验法则

```python
# 数据点数量 vs 推荐的 knn
数据点 < 50:    knn = 3-5
数据点 50-200:  knn = 5-10  (默认 5)
数据点 200-500: knn = 10-15
数据点 > 500:   knn = 15-20
```

对于你的胚胎数据：
- 每个胚胎有不同数量的时间点（frames）
- 通常 50-500 个时间点
- **knn = 5** 是合理的默认值 ✅

## 🧪 实验建议

如果想调整 knn：

```bash
# 使用较小的 knn（更关注局部）
python3 generate_tphate_for_aadhitya.py \
    --knn 3 \
    ...其他参数...

# 使用较大的 knn（更关注全局）
python3 generate_tphate_for_aadhitya.py \
    --knn 10 \
    ...其他参数...
```

## 📚 技术细节

在 T-PHATE 中：
- knn 用于构建 **k-nearest neighbor graph**
- 这个图决定了数据的局部邻域结构
- 影响最终的嵌入（embedding）结果
- 通常 knn 应该远小于总数据点数（例如 < 10%）

## ✅ 总结

- **knn = 5** 是默认值，通常工作良好
- 对于胚胎轨迹数据，5-10 是合理范围
- 不需要经常调整，除非：
  - 结果看起来不对
  - 计算时间太长（减少 knn）
  - 丢失了重要结构（增加 knn）
